## 3. Complete Syllabus (120 Hours)

This syllabus is what the **best DevOps training in Electronic City** programs follow in 2026.

### Module 1: Linux & Scripting (20 hrs)
- Bash commands, file permissions, processes, cron jobs
- Python basics for automation (loops, functions, file I/O)
- **Practice:** Write a script to rotate logs and email alerts

### Module 2: Version Control (15 hrs)
- Git workflow (clone, commit, branch, merge, rebase)
- GitHub/GitLab: pull requests, code reviews, actions
- **Practice:** Resolve a merge conflict and open a PR

### Module 3: CI/CD Pipelines (25 hrs)
- Jenkins (declarative pipelines, shared libraries)
- GitLab CI, GitHub Actions
- **Practice:** Build a pipeline that tests, builds, and deploys a Node.js app to EC2

### Module 4: Config Mgmt & IaC (20 hrs)
- Ansible (playbooks, roles, vault, dynamic inventories)
- Terraform (state, modules, remote backends)
- **Practice:** Use Terraform to provision VPC + EC2, then configure with Ansible

### Module 5: Containers & Orchestration (25 hrs)
- Docker (Dockerfile, compose, networking, volumes, registry)
- Kubernetes (pods, deployments, services, ingress, configmaps, secrets, Helm)
- **Practice:** Deploy a multi-container app on Minikube with rolling update

### Module 6: Cloud Platforms (15 hrs)
- AWS: EC2, S3, VPC, IAM, RDS, EKS, CloudWatch
- **Practice:** Set up auto-scaling group with load balancer

### Module 7: Monitoring & Logging (10 hrs)
- Prometheus (metrics, Alertmanager)
- Grafana (dashboards, data sources)
- ELK Stack basics (Elasticsearch, Logstash, Kibana)
- **Practice:** Monitor a K8s cluster and set CPU alert

### Module 8: DevSecOps & AIOps (10 hrs – 2026 update)
- SAST/DAST (SonarQube, Snyk)
- Security scanning in CI/CD pipelines
- GitOps with ArgoCD
- AI-assisted monitoring basics
- **Practice:** Add a security scan stage to your Jenkins pipeline

## 8. 3-Step Verification Process Before Enrolling

### Step 1: Attend a Live Demo Class
- Not a recorded video. Watch the instructor debug a broken pipeline in real time.
- Ask: *“Can every student type along on their own cloud console?”*

### Step 2: Call Past Students
- Request 3 alumni contacts. Call them and ask:
  - *“Were you placed? How long did it take?”*
  - *“What was your starting salary?”*
  - *“Would you recommend this institute?”*

### Step 3: Visit the Physical Lab
- See the infrastructure. Ask to see a current student’s cloud console.
- Check if the lab has enough computers and internet speed.

**If an institute refuses any of these steps, walk away.**
